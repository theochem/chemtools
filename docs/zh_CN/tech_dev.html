开发者指南¶

提交错误¶

如果您在使用ChemTools的过程中遇到了程序出错的情况，欢迎您到我们的 问题页面 提交相关的报告或反馈。

安装ChemTools¶

我们使用Github来管理ChemTools的开发，您可以用下列命令下载我们最新的 代码

git clone https://github.com/QuantumElephant/chemtools.git chemtools

对于开发者来说，我们推荐就地安装ChemTools。在安装前，请确保你已经安装好了所有必须的 依赖库 。您可以用下列命令就地安装ChemTools：

cd chemtools
python setup.py build

您需要添加 PYTHONPATH 和 CTDATA 路径到环境变量中，~/.bashrc (Linux) or ~/.profile (Mac OS):

export PYTHONPATH=$PYTHONPATH:{path_to_chemtools_repo}
export CTDATA={path_to_chemtools_repo}/data

运行测试¶

为确保ChemTools各部分正常运行，我们建议您运行 nosetests 来测试：

nosetests -v chemtools

在测试的过程中，一些警告信息可能会显示在屏幕上，不同担心，这些是正常情况。需要留意的是在测试结束后，正常情况下不应该有任何错误或者测试失败的信息。

贡献代码¶

我们推荐您使用`Git`来参与开发，Fork我们在Github上的Repo，修改，保存，然后再在Github上通过 Pull request 来提交你所做的修改

  • Fork我们在Github上面的 Repo :点击网页右上角 Fork 按钮。这样，在您自己的账号下就会有一份完全一摸一样的ChemTools的拷贝。

  • 将拷贝的Repo添加到你本地电脑中的远程Repo中：

    git remote add your_remote_name git@github.com:YourAccount/chemtools.git

  • 在本地电脑上创建一个新的分支(Branch):

    git checkout -b your_feature_branch_name

  • 将您的名字和邮箱保存在ChemTools根目录中的 AUTHORS 文件中。请确保您输入的名字和您Git所设置的信息相匹配。您可以用下列命令查看相关的Git信息：

    git config user.name
git config user.email

  • 在新创建的分支上对代码进行修改，当您完成编辑之后，使用以下命令：

    git add files_you_modified
git commit

    在Commit的文本中写下你所做的修改并保存。

  • 将你修改后的代码推送（Push）到你自己的远程Repo中：

    git push your_remote_name your_feature_branch_name

  • 最后，回到您自己Fork的Github的网页，点击 Pull request 提交你所做的修改。所有提交的修改都会在通过ChemTools的代码测试之后被审阅。你可以在 Pull request 页面查看测试结果，并修正可能生成的错误。

创建文档¶

如果你想要从源代码编译生成说明文档，那么下列软件库则是必不可少的：

  • Sphinx >=1.3.1: http://sphinx.pocoo.org/
  • sphinxcontrib-bibtex >= 0.3.5: https://pypi.python.org/pypi/sphinxcontrib-bibtex
  • IPython >= 3.2.1: https://ipython.org/install.html

根据你的电脑系统，按步骤安装这些依赖库

  • Ubuntu Linux 16.04

    sudo apt-get install python-sphinx ipython
pip install --user --upgrade sphinxcontrib-bibtex

  • Ubuntu Linux 15.04 & 14.04

    pip install --user --upgrade sphinx sphinxcontrib-bibtex ipython

  • Mac OS

    sudo port install py27-sphinx py27-ipython
pip install --user --upgrade sphinxcontrib-bibtex

您可以用Git来下载ChemTools订制版的Sphinx RTD主题：

git submodule update --init doc/_themes/sphinx_rtd_theme

同时，请确认正确设置了 环境变量 CTDATA ；顺利下载了 实例文件。

您可以使用下列命令，自动生成软件API文档和HTML网页文档：

cd doc
make clean
make html

您可以通过直接点击打开 _build/html/index.html 文件 或是通过下列命令打开查看生成的网页：

open _build/html/index.html

如果上述命令在您的电脑系统上无效，如在Ubuntu 16.04等，你也可以尝试：

see _build/html/index.html

质量保证¶

每当您使用Pull request推送你的代码到ChemTools Repo的时候，远程服务器都会对所推送的代码进行测试，以确保所有上传的代码符合ChemTools的质量要求。如果您想在本地电脑上运行普通测试，请查阅 测试 页面；如果您想要在本地电脑上运行质量保证测试,请首先在电脑上安装下列依赖库：

  • PyLint >= 1.5.0: https://www.pylint.org/
  • pycodestyle >= 2.0.0: http://pycodestyle.readthedocs.io/
  • pydocstyle >= 1.0.0: http://pydocstyle.readthedocs.io/
  • coverage >= 4.1: https://coverage.readthedocs.io/
  • Git >= 1.8: https://git-scm.com/
  • GitPython >= 2.0.5: http://gitpython.readthedocs.io/

使用Git 下载 质量保证 子模版：

git submodule update --init tools/inspector

之后，利用该模版中的bash脚本对pre-commit hooks进行设置，再依次运行质量保证的测试脚本：

# it is installed in the relative path
cd tools/inspector
./install.sh
cd ../..

在这个阶段，质量保证测试可以在ChemTools的根目录下进行。例如，测试 pylint 可以使用如下命令，

# from ChemTools parent directory
./tools/inspector/qa/simulate_trapdoor_pr.py tools/inspector/qa/trapdoor_pylint.py

用下列命令运行所有测试脚本，

# from ChemTools parent directory
for i in tools/inspector/qa/trapdoor_*; do tools/inspector/qa/simulate_trapdoor_pr.py $i; done

注意，在开发的时候，您应该在特征（feature）分支上，而不是主分支（master）上进行。在开发完成之后再合并（Merge／rebase）至最新的主分支上。同时在测试之前，也请确保该分支下没有未保存的编辑内容。
